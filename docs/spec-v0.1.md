# VRChatアバター連携スマートフォンシステム 基本仕様書

**文書バージョン:** 0.1
**作成日:** 2026年7月13日
**仮称:** Avatar Smartphone Link
**対象:** VRChat Avatars 3.0/PC版VRChat/iOS・Androidスマートフォン

> **注意:** 本文書は受領した仕様書 v0.1 の原文である。実装にあたって確定・修正した事項は
> [spec-errata.md](spec-errata.md) および [protocol.md](protocol.md) を正とする。

---

## 1. 目的

本システムは、現実のスマートフォンからVRChatアバターに搭載されたスマートフォンギミックを操作し、以下を実現する。

* 実スマートフォンからアバター上のスマートフォンを表示・非表示にする
* アバター上の画面を切り替える
* 通話、通知、撮影、音楽再生などの状態を演出する
* VRChat側で変更された状態を実スマートフォンへ反映する
* アバターを見ている他プレイヤーにも主要な状態を表示する
* 通信切断時に安全な表示へ移行する

本システムは、実スマートフォンのOSやアプリをVRChat内で直接動作させるものではない。スマートフォンアプリから状態値を送信し、アバターのAnimatorと事前作成済み画面素材によって、スマートフォンの動作を疑似的に表現する。

---

## 2. 前提条件

### 2.1 動作環境

* VRChatはPC版を使用する
* アバターはAvatars 3.0形式でアップロードする
* PCとスマートフォンは原則として同一LANに接続する
* VRChatのOSC機能を有効にする
* PC上で専用中継アプリを起動する
* スマートフォン上で専用コンパニオンアプリを起動する

VRChatのOSCはAction Menuの`OSC > Enabled`から有効化でき、OSC Avatar Parametersを利用してカスタムアバターパラメータを外部アプリから操作できる。

### 2.2 VRChat側の制約

VRChatアバターでは任意のカスタムスクリプトを実行できないため、アバター内の処理はAnimator、Animation Clip、Expression Parameters、VRC Contact、VRC Constraintなど、許可された機能だけで構成する。許可リスト外のコンポーネントやカスタムスクリプトは動作せず、アップロードを妨げる場合がある。

OSC Avatar Parametersで扱える値は、基本的に以下の3種類とする。

* `Bool`
* `Int`
* `Float`

OSC設定では、入力・出力値として`Int`、`Bool`、`Float`がサポートされている。

## 3. システム構成

### 3.1 推奨構成

```text
実スマートフォン
  │
  │ WebSocket通信
  │ 状態変更・応答・ペアリング
  ▼
PC中継アプリ
  │
  │ OSC/OSCQuery
  ▼
PC版VRChat
  │
  │ Avatar Parameters
  ▼
アバターFX Animator
  │
  ▼
アバター上のスマートフォンギミック
```

### 3.2 各構成要素

#### A. スマートフォンアプリ

利用者が直接操作するアプリケーション。

主な役割:

* PC中継アプリとのペアリング
* 接続状態の表示
* アバターの対応状況表示
* スマートフォンギミックの操作
* VRChat側状態の受信
* バッテリー状態など、許可された端末情報の送信

#### B. PC中継アプリ

スマートフォンアプリとVRChatの間を接続する常駐アプリケーション。

主な役割:

* スマートフォンとの認証
* WebSocketサーバー
* OSCおよびOSCQueryクライアント
* Avatar Parameterの送受信
* Avatar IDの検出
* 接続監視
* パラメータ値の検証
* 再接続処理
* ログ記録

VRChatはOSCQueryに対応しており、対応アプリ同士の検出や機能情報の取得、自動設定に利用できる。中継アプリはOSCQueryを優先し、固定ポートを代替手段として使用する。

#### C. VRChatアバターギミック

VRChat SDKとUnity Animatorで作成する。

主な役割:

* スマートフォン本体の表示
* 画面の切り替え
* 手持ち、通話、自撮りなどのポーズ
* 接続状態表示
* 通知演出
* バッテリー表示
* 通話・メディア状態表示

## 4. 通信方式

### 4.1 スマートフォン-PC中継アプリ間

| 項目         | 仕様             |
| ---------- | -------------- |
| プロトコル      | WebSocket      |
| ネットワーク     | 同一LAN          |
| データ形式      | UTF-8 JSON     |
| 接続方向       | スマートフォンからPCへ接続 |
| 認証         | ワンタイムペアリングトークン |
| 再接続        | 自動             |
| ハートビート     | 2秒間隔           |
| 切断判定       | 6秒間応答なし        |
| プロトコルバージョン | 整数で管理          |

### 4.2 PC中継アプリ-VRChat間

優先順位は以下とする。

1. OSCQueryによる自動検出
2. ローカルホスト固定ポート
3. 利用者による手動ポート設定

VRChatの標準設定では、外部からのOSC受信にポート`9000`、VRChatからの送信にポート`9001`が使用される。起動引数により変更可能である。

固定ポート使用時の標準設定:

```text
VRChat受信先:
127.0.0.1:9000

中継アプリ受信先:
127.0.0.1:9001
```

スマートフォンからVRChatへ直接OSCを送る構成は、代替モードとしてのみ扱う。標準構成では、VRChatとのOSC通信をPC内に限定する。

## 5. ペアリング仕様

### 5.1 初回ペアリング

1. PC中継アプリを起動する
2. 中継アプリがペアリング用QRコードを表示する
3. スマートフォンアプリでQRコードを読み取る
4. スマートフォンが中継アプリへ接続要求を送る
5. 中継アプリが接続を許可する
6. 双方が端末識別情報を保存する
7. VRChatとの接続確認を行う

QRコードには以下を含める。

```json
{
  "protocol": 1,
  "host": "192.168.1.10",
  "port": 27810,
  "token": "一時的なランダムトークン"
}
```

### 5.2 ペアリングトークン

* 128ビット以上のランダム値とする
* 有効期限は5分とする
* 一度使用したトークンは無効化する
* ペアリング解除時に保存済み認証情報を削除する
* 利用者が明示的にペアリングモードを開始した場合のみ新規端末を許可する

### 5.3 再接続

保存済み端末は、同一LAN内でPC中継アプリを発見した場合、自動再接続を試行する。

再接続間隔:

```text
1秒
2秒
5秒
10秒
以降10秒間隔
```

## 7. VRChatアバターパラメータ仕様

### 7.1 命名規則

パラメータ名の接頭辞は`Phone/`とする。

OSCアドレスは以下の形式になる。

```text
/avatar/parameters/Phone/Visible
/avatar/parameters/Phone/Page
/avatar/parameters/Phone/Pose
```

VRChatでは、`/avatar/parameters/パラメータ名`に値を送信すると、同名のAvatar Parameterを操作できる。

### 7.2 パラメータ一覧

| パラメータ               | 型    |   初期値 | Synced | Saved | 内容         |
| ------------------- | ---- | ----: | ------ | ----- | ---------- |
| `Phone/Visible`     | Bool | false | ON     | ON    | 本体表示       |
| `Phone/Connected`   | Bool | false | ON     | OFF   | 実スマホ接続状態   |
| `Phone/Locked`      | Bool |  true | ON     | ON    | ロック状態      |
| `Phone/Page`        | Int  |     0 | ON     | ON    | 表示画面       |
| `Phone/Pose`        | Int  |     0 | ON     | ON    | 保持ポーズ      |
| `Phone/Battery`     | Int  |    10 | ON     | OFF   | バッテリー段階    |
| `Phone/CallState`   | Int  |     0 | ON     | OFF   | 通話状態       |
| `Phone/MediaState`  | Int  |     0 | ON     | OFF   | メディア状態     |
| `Phone/NotifyType`  | Int  |     0 | ON     | OFF   | 通知演出種別     |
| `Phone/EventToggle` | Bool | false | ON     | OFF   | イベント発火用反転値 |

同期パラメータの使用量は以下となる。

```text
Bool × 4 = 4ビット
Int × 6  = 48ビット
合計      = 52ビット
```

VRChatの同期カスタムパラメータ容量は最大256ビットで、`Int`と`Float`は各8ビット、`Bool`は1ビットを使用する。したがって本仕様は、他のアバターギミック用に204ビットを残す。

### 7.3 Page値

|     値 | 画面    |
| ----: | ----- |
|     0 | ロック画面 |
|     1 | ホーム   |
|     2 | 通知    |
|     3 | 通話    |
|     4 | カメラ   |
|     5 | メディア  |
|     6 | 設定    |
|     7 | 接続エラー |
| 8~255 | 将来拡張用 |

### 7.4 Pose値

|     値 | ポーズ   |
| ----: | ----- |
|     0 | 収納    |
|     1 | 右手持ち  |
|     2 | 左手持ち  |
|     3 | 通話    |
|     4 | 自撮り   |
|     5 | 両手操作  |
| 6~255 | 将来拡張用 |

### 7.5 CallState値

|  値 | 状態     |
| -: | ------ |
|  0 | 通話なし   |
|  1 | 着信中    |
|  2 | 発信中    |
|  3 | 通話中    |
|  4 | 通話終了演出 |

### 7.6 MediaState値

|  値 | 状態      |
| -: | ------- |
|  0 | 停止      |
|  1 | 再生      |
|  2 | 一時停止    |
|  3 | 次トラック演出 |
|  4 | 前トラック演出 |

### 7.7 Battery値

バッテリー残量は0~10の11段階で送信する。

```text
0  = 0~4%
1  = 5~14%
2  = 15~24%
...
9  = 85~94%
10 = 95~100%
```

アバター側では11種類の事前作成済み表示を切り替える。

## 8. VRChat Animator仕様

### 8.1 FXレイヤー構成

```text
Phone_Visibility
Phone_Page
Phone_Connection
Phone_Battery
Phone_Call
Phone_Media
Phone_Notification
Phone_Effects
```

### 8.2 Visibilityレイヤー

`Phone/Visible`に応じて以下を切り替える。

```text
false:
スマートフォン本体 非表示
画面 非表示
接続ランプ 非表示

true:
スマートフォン本体 表示
選択画面 表示
接続状態 表示
```

### 8.3 Pageレイヤー

`Phone/Page`の値に応じて画面を切り替える。

実装方式は以下のいずれかとする。

* 画面ごとにGameObjectを分けて表示切り替え
* テクスチャアトラスのUV切り替え
* マテリアルプロパティのアニメーション

MVPでは、実装とデバッグが容易なGameObject切り替え方式を採用する。

### 8.4 イベント演出

`Phone/EventToggle`は、イベント発生ごとに値を反転する。

```text
1回目の通知: false → true
2回目の通知: true → false
3回目の通知: false → true
```

Animatorにはtrue側とfalse側の両方に同一の通知演出を用意し、どちらへ遷移しても演出が再生されるようにする。

`Trigger`型は使用しない。VRChatは、状態表現には`Int`、`Float`、`Bool`を使用することを推奨しており、Triggerはリモート表示やアバターバージョン間で同期ずれが起こる可能性がある。

### 8.5 ポーズ制御

スマートフォンの追従先は以下とする。

* 右手基準点
* 左手基準点
* 頭部通話基準点
* 自撮り基準点
* 腰収納基準点

追従処理にはVRC Constraintを使用する。

ポーズに身体アニメーションを含める場合は、GestureまたはActionレイヤーを使用し、FXレイヤーからHumanoidボーンを直接操作しない。

## 9. 画面表示仕様

### 9.1 基本方針

アバターの画面は事前作成済み素材の切り替えによって表現する。

以下には対応しない。

* 実スマートフォン画面のミラーリング
* 動画ストリーミング
* 任意画像の転送
* 任意文字列の表示
* Webページの表示
* 実際のスマートフォンアプリの実行

VRChatのAvatar Parametersから受け取れるデータは数値・真偽値に限定され、アバター内では任意のカスタムスクリプトも使用できない。このため、実画面や任意データの直接転送ではなく、番号に対応する事前作成素材を表示する方式とする。

### 9.2 他プレイヤーへの同期

以下はSynced Parameterとして他プレイヤーへ同期する。

* スマートフォン表示状態
* 現在の画面
* ポーズ
* 接続状態
* バッテリー段階
* 通話状態
* メディア状態
* 通知演出

カスタム同期パラメータは通常Playable同期となり、必要に応じて毎秒1~10回程度更新されるため、高速な連続操作やリアルタイム映像用途には使用しない。

イベント送信は原則として1秒に1回以下に制限する。

| -------------------- | ------------- |

| `INVALID_VALUE`      | 値が範囲外         |
| `PAIRING_REQUIRED`   | ペアリングが必要      |
| `AUTH_FAILED`        | 認証失敗          |
| `RATE_LIMITED`       | 送信頻度超過        |
| `TIMEOUT`            | VRChat側応答なし   |

## 11. Avatar検出仕様

VRChatは、ローカルユーザーがアバターを変更すると、OSCが有効な場合に`/avatar/change`へAvatar IDを出力する。中継アプリはこのメッセージを監視する。

アバター変更時の処理:

1. `/avatar/change`を受信する
2. Avatar IDを保存する
3. OSCQueryまたはパラメータ出力から対応パラメータを確認する
4. `Phone/Visible`と`Phone/Page`の存在を確認する
5. 対応している場合、スマートフォンへ全状態を送る
6. 非対応の場合、スマートフォンの操作UIを無効化する
7. 前アバター向けの未処理コマンドを破棄する

公開済みアバターでは、VRChatがAvatar IDごとのOSC設定ファイルを生成し、入力・出力アドレスを管理する。Build & Testだけでは設定ファイルが保存されないため、最終結合試験では公開済みのテストアバターを使用する。

## 12. 状態同期仕様

### 12.1 スマートフォンからVRChat

1. スマートフォンが変更要求を送信
2. 中継アプリが型と範囲を検証
3. 中継アプリがOSCメッセージを送信
4. VRChatがAvatar Parameterを変更
5. VRChatから同パラメータの出力を受信
6. 中継アプリがスマートフォンへ確定値を返す

### 12.2 VRChatからスマートフォン

Expression MenuやContactで値が変更された場合、中継アプリはVRChatから出力された変更値をスマートフォンへ送る。

Expression Parametersは、Expressions Menu、Avatar Parameter Driver、OSCから操作できる。

### 12.3 状態の正本

接続中のパラメータ値については、VRChatから最後に出力された値を正本とする。

スマートフォンは操作直後に仮表示を行えるが、VRChatから確定値が返った時点で表示を修正する。

### 12.4 タイムアウト

変更要求から1.5秒以内にVRChat側の値を確認できない場合、以下を返す。

```text
status: timeout
```

その後もVRChatから値を受信した場合は、通常の状態更新としてスマートフォンへ送信する。

## 13. 接続切断時の動作

### 13.1 スマートフォン切断

6秒間ハートビートを受信できなかった場合:

```text
Phone/Connected = false
```

以下の値は維持する。

* `Phone/Visible`
* `Phone/Page`
* `Phone/Pose`
* `Phone/Locked`

通話・メディアなど一時状態はリセットする。

```text
Phone/CallState = 0
Phone/MediaState = 0
Phone/NotifyType = 0
```

### 13.2 VRChat切断

VRChatまたはOSCを検出できなくなった場合:

* スマートフォンへ`VRCHAT_NOT_FOUND`を通知
* 操作ボタンを無効化
* 中継アプリは再検出を継続
* VRChat再検出後に全状態を再取得

### 13.3 中継アプリ終了

スマートフォンは切断状態へ移行し、自動再接続を試行する。

## 14. セキュリティ・プライバシー

### 14.1 セキュリティ要件

* インターネット経由の接続は初期版では許可しない
* 同一LANからのみ接続を受け付ける
* ペアリング済み端末以外は操作できない
* OSC通信はPC内のループバック通信に限定する
* 未定義パラメータはVRChatへ転送しない
* Int、Float、Bool以外の値を拒否する
* 値が定義範囲外の場合は拒否またはクランプする
* 連続送信にレート制限を設ける
* 認証失敗をログに記録する
* ログへ認証トークンを出力しない

### 14.2 プライバシー要件

初期版では以下を収集・送信しない。

* 通知本文
* メッセージ内容
* 連絡先
* 通話相手
* 写真
* 音声
* 位置情報
* VRChat認証情報
* VRChatのログインCookie

保存可能な情報:

* ペアリング済みPC識別子
* アプリ設定
* 最後に選択した画面
* バッテリー連携の有効・無効
* 匿名化したエラーログ

## 15. 非機能要件

### 15.1 応答性能

| 項目                 |      目標 |
| ------------------ | ------: |
| スマートフォン操作から中継アプリ受付 | 100ms以内 |
| ローカルアバター反映         | 250ms以内 |
| スマートフォンへの確定応答      |  1.5秒以内 |
| 他プレイヤーへの安定状態反映     |    2秒以内 |
| 切断検出               |    6秒以内 |

他プレイヤーへの反映速度はVRChatのPlayable Parameter同期に依存し、高速同期を保証しない。

### 15.2 可用性

* 中継アプリはVRChatより先に起動しても動作する
* VRChatの再起動後に自動再接続する
* スマートフォンのWi-Fi再接続後に自動復帰する
* アバター変更後に操作対象を自動更新する
* 不正パケットを受信してもアプリが終了しない

### 15.3 ログ

PC中継アプリは以下を記録する。

* 起動・終了
* VRChat検出
* OSC有効・無効
* Avatar ID変更
* スマートフォン接続・切断
* ペアリング
* パラメータ検証エラー
* 通信タイムアウト
* 予期しない例外

ログレベル:

```text
ERROR
WARN
INFO
DEBUG
```

OSC値の全件記録はDEBUG時のみとする。

## 16. テスト仕様

### 16.1 基本接続試験

| ID    | 試験内容       | 合格条件             |
| ----- | ---------- | ---------------- |

| T-003 | 未認証端末接続    | 接続を拒否する          |

| T-005 | OSC無効      | OSC無効エラーを表示する    |

### 16.2 パラメータ試験

| ID    | 試験内容          | 合格条件             |
| ----- | ------------- | ---------------- |
| T-101 | Visible変更     | 本体表示が切り替わる       |
| T-102 | Page変更        | 指定した画面へ切り替わる     |
| T-103 | Pose変更        | 指定した保持位置へ移動する    |
| T-104 | Battery変更     | 対応する残量表示になる      |
| T-105 | CallState変更   | 通話状態が反映される       |
| T-106 | MediaState変更  | 再生状態が反映される       |
| T-107 | EventToggle反転 | 毎回通知演出が再生される     |
| T-108 | 範囲外値          | 拒否または仕様値にクランプされる |

### 16.3 双方向同期試験

| ID    | 試験内容                | 合格条件              |
| ----- | ------------------- | ----------------- |
| T-201 | スマホから画面変更           | VRChatとスマホ表示が一致する |
| T-202 | Expression Menuから変更 | スマホへ変更が反映される      |
| T-203 | Contactから変更         | スマホへ変更が反映される      |
| T-204 | 同時操作                | VRChatの確定値へ収束する   |

### 16.4 切断・復旧試験

| ID    | 試験内容       | 合格条件         |
| ----- | ---------- | ------------ |
| T-301 | Wi-Fi切断    | 6秒以内に切断表示となる |
| T-302 | Wi-Fi復旧    | 自動再接続する      |
| T-303 | 中継アプリ再起動   | スマホが再接続する    |
| T-304 | VRChat再起動  | OSC再検出後に復旧する |
| T-305 | 対応外アバターへ変更 | 操作UIが無効になる   |
| T-306 | 対応アバターへ戻す  | 状態を再取得して復旧する |

### 16.5 リモート表示試験

別ユーザーから以下を確認する。

* スマートフォンの表示状態
* 画面
* ポーズ
* バッテリー表示
* 通話状態
* 通知演出
* 接続状態

連続操作ではなく、各操作の間隔を1秒以上空けて試験する。
