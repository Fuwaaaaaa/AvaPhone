# 仕様書 v0.1 からの確定・修正事項

実装計画策定時(2026-07-13)に検出した仕様書の不足・不整合と、その確定内容。
本文書と [protocol.md](protocol.md) が仕様書 v0.1 に優先する。

## E-1: FXレイヤーに `Phone_Pose` を追加(9レイヤー構成)

仕様書 8.1 のFXレイヤー一覧(8レイヤー)には、`Phone/Pose` の値に応じて
VRC Constraint のソースWeightを切り替えるレイヤーが存在しない。
ポーズ切替(仕様書 8.5)の実現に必須のため、9層目として `Phone_Pose` を追加する。

## E-2: `Phone/NotifyType` の値域を 0-4 に確定

仕様書 7.2 で `Phone/NotifyType` の値域が未定義だった。MVPでは 0-4 とする
(0 = 演出なし、1-4 = 通知演出種別)。中継アプリの検証ルール、
アバター側の通知オーバーレイ数、スマホアプリのUIすべてをこの値域に統一する。

## E-3: `Phone/Locked` はMVPでは専用FXレイヤーを持たない

仕様書のFXレイヤー一覧にLocked用レイヤーがないため、MVPでは
パラメータ+Expression Menuトグルのみとし、ロック画面の表示は `Phone/Page = 0` で表現する。
将来、画面上のロックアイコン表示などに使用できるようパラメータ自体は維持する。

## E-4: 認証ハンドシェイクのメッセージ形式を確定

仕様書 10章 にはペアリング後の認証メッセージの形式が定義されていなかった。
WebSocket接続後の最初のメッセージは `auth` 必須とする。詳細は [protocol.md](protocol.md) 参照。

## E-5: イベント操作は専用メッセージを設けない

仕様書 6.2 の「イベント操作」(通知演出・着信・シャッター等)は、すべて既存の
`parameter.set` で表現する(通知演出 = `Phone/EventToggle` の反転、着信 = `Phone/CallState` の変更等)。
`Phone/EventToggle` の反転はスマートフォンアプリが現在値を反転して送信する。
